This guide will walk you through making your first print request with Print My Bridge in just a few minutes.

## Prerequisites

Before you begin, ensure you have:

- Print My Bridge installed and running (see [Installation](/installation))
- A printer connected to your system
- curl or a similar HTTP client (for testing)

<Note>
You can verify Print My Bridge is running by checking for the application icon in your system tray.
</Note>

## Generate an API token

All API requests to Print My Bridge require authentication with a bearer token.

<Steps>
  <Step title="Open Print My Bridge">
    Click the Print My Bridge icon in your system tray or launch the application from your applications menu.
  </Step>
  
  <Step title="Generate a new token">
    In the main window, click the **Generate New Token** button. A secure 32-character token will be generated for you.
  </Step>
  
  <Step title="Copy the token">
    Click the **Copy** button next to the token to copy it to your clipboard. The token will look something like:
    
    ```
    AbCdEfGh1234567890XyZaBcDeFgH12
    ```
    
    <Warning>
    Keep your token secure! Anyone with this token can send print jobs to your printers. Never commit tokens to version control or share them publicly.
    </Warning>
  </Step>
</Steps>

## Verify the server is running

Before making authenticated requests, verify the HTTP server is running by checking the health endpoint:

```bash
curl http://localhost:8765/health
```

<CodeGroup>
```json Response
{
  "status": "ok",
  "service": "print-my-bridge",
  "version": "0.1.0"
}
```
</CodeGroup>

<Tip>
The `/health` endpoint doesn't require authentication, making it perfect for monitoring whether Print My Bridge is running.
</Tip>

## List available printers

Now let's use your API token to list all printers connected to your system:

```bash
curl -H "x-api-token: YOUR_TOKEN" \
  http://localhost:8765/api/printers
```

Replace `YOUR_TOKEN` with the token you generated. You'll receive a response like:

<CodeGroup>
```json Response
{
  "printers": [
    {
      "name": "HP LaserJet Pro",
      "status": "ready",
      "is_default": true,
      "supports_color": true,
      "paper_sizes": ["A4", "Letter", "Legal"]
    },
    {
      "name": "Canon PIXMA",
      "status": "ready",
      "is_default": false,
      "supports_color": true,
      "paper_sizes": ["A4", "Letter", "4x6"]
    }
  ]
}
```
</CodeGroup>

<Note>
If you don't see any printers, verify that you have printers installed on your system and that they are online.
</Note>

## Make your first print request

Now you're ready to print! The Print My Bridge API accepts print jobs with different content types.

### Print a text document

Let's start with a simple text document:

```bash
curl -X POST \
  -H "x-api-token: YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "content": "Hello from Print My Bridge!\n\nThis is a test print job.",
    "content_type": "text",
    "copies": 1
  }' \
  http://localhost:8765/api/print
```

For text content, you can send plain text directly in the `content` field.

<CodeGroup>
```json Success Response
{
  "success": true,
  "message": "Document sent to printer successfully",
  "job_id": "12345"
}
```

```json Error Response
{
  "success": false,
  "message": "Printer not found"
}
```
</CodeGroup>

### Print to a specific printer

To print to a specific printer instead of the default, add the `printer_name` field:

```bash
curl -X POST \
  -H "x-api-token: YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "content": "Hello from Print My Bridge!",
    "content_type": "text",
    "printer_name": "Canon PIXMA",
    "copies": 1
  }' \
  http://localhost:8765/api/print
```

### Print multiple copies

To print multiple copies, set the `copies` field:

```bash
curl -X POST \
  -H "x-api-token: YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "content": "Hello from Print My Bridge!",
    "content_type": "text",
    "copies": 3
  }' \
  http://localhost:8765/api/print
```

## Encode content for PDF and image printing

For PDF and image content, you need to base64-encode the file data. Here's how to encode files:

<CodeGroup>
```bash Linux/macOS
# Encode a file to base64
base64 -w 0 document.pdf

# Or encode and print directly
curl -X POST \
  -H "x-api-token: YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d "{
    \"content\": \"$(base64 -w 0 document.pdf)\",
    \"content_type\": \"pdf\"
  }" \
  http://localhost:8765/api/print
```

```powershell Windows (PowerShell)
# Encode a file to base64
[Convert]::ToBase64String([IO.File]::ReadAllBytes("document.pdf"))

# Or use certutil
certutil -encode document.pdf document.b64
```

```python Python
import base64

# Encode a file
with open('document.pdf', 'rb') as f:
    encoded = base64.b64encode(f.read()).decode('utf-8')
    print(encoded)
```

```javascript JavaScript/Node.js
const fs = require('fs');

// Encode a file
const fileBuffer = fs.readFileSync('document.pdf');
const encoded = fileBuffer.toString('base64');
console.log(encoded);
```
</CodeGroup>

## Complete example with error handling

Here's a complete example in Python that includes error handling:

```python print_example.py
import requests
import base64
import json

# Configuration
API_URL = 'http://localhost:8765'
API_TOKEN = 'YOUR_TOKEN_HERE'

def print_document(file_path, content_type, printer_name=None, copies=1):
    """Send a print job to Print My Bridge"""
    
    # Read and encode the file
    with open(file_path, 'rb') as f:
        content = base64.b64encode(f.read()).decode('utf-8')
    
    # Prepare the request
    headers = {
        'x-api-token': API_TOKEN,
        'Content-Type': 'application/json'
    }
    
    payload = {
        'content': content,
        'content_type': content_type,
        'copies': copies
    }
    
    if printer_name:
        payload['printer_name'] = printer_name
    
    # Send the print request
    try:
        response = requests.post(
            f'{API_URL}/api/print',
            headers=headers,
            json=payload
        )
        response.raise_for_status()
        
        result = response.json()
        if result.get('success'):
            print(f"✓ Print job submitted: {result.get('job_id')}")
            return True
        else:
            print(f"✗ Print failed: {result.get('message')}")
            return False
            
    except requests.exceptions.RequestException as e:
        print(f"✗ Request failed: {e}")
        return False

# Example usage
if __name__ == '__main__':
    # Print a PDF document
    print_document('document.pdf', 'pdf', copies=1)
    
    # Print to a specific printer
    print_document('receipt.txt', 'text', printer_name='Receipt Printer')
```

<Tip>
For production use, store your API token in environment variables instead of hardcoding it:

```python
import os
API_TOKEN = os.environ.get('PRINT_MY_BRIDGE_TOKEN')
```
</Tip>

## Troubleshooting

If you encounter issues during the quick start:

### Connection refused

```bash
curl: (7) Failed to connect to localhost port 8765: Connection refused
```

**Solution**: Ensure Print My Bridge is running. Check your system tray for the application icon.

### Unauthorized error

```json
{
  "error": "Unauthorized"
}
```

**Solution**: Verify you're including the correct token in the `x-api-token` header.

### Rate limit exceeded

```json
{
  "error": "Rate limit exceeded"
}
```

**Solution**: You've exceeded the rate limit (default: 60 requests/minute). Wait a minute and try again, or increase the limit in the configuration file.

### Unsupported format

```json
{
  "success": false,
  "message": "Unsupported format"
}
```

**Solution**: Ensure the `content_type` is one of the allowed types (`pdf`, `html`, `text`, `image`). Check your configuration file to see which types are enabled.

### Printer not found

```json
{
  "success": false,
  "message": "Printer not found"
}
```

**Solution**: Verify the printer name exactly matches one of the printers returned by `/api/printers`. Printer names are case-sensitive.
