## Overview

Building Print My Bridge from source allows you to customize the application, contribute to development, or create custom builds for your specific needs. This guide covers the complete build process for all supported platforms.

## Prerequisites

Before building Print My Bridge, ensure you have the following tools installed:

### Required Tools

<Steps>
  <Step title="Install Rust">
    Print My Bridge requires **Rust 1.70 or higher**.

    <CodeGroup>
    ```bash macOS/Linux
    curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
    ```

    ```powershell Windows
    # Download and run rustup-init.exe from https://rustup.rs/
    ```
    </CodeGroup>

    Verify the installation:
    ```bash
    rustc --version
    cargo --version
    ```
  </Step>

  <Step title="Install Node.js (Optional)">
    While not strictly required for the current build, **Node.js 16+** may be needed for future frontend development.

    Download from [nodejs.org](https://nodejs.org/) or use a version manager:

    <CodeGroup>
    ```bash macOS (Homebrew)
    brew install node
    ```

    ```bash Linux (nvm)
    curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
    nvm install 16
    ```

    ```powershell Windows
    # Download installer from nodejs.org
    ```
    </CodeGroup>
  </Step>

  <Step title="Install Tauri CLI">
    The Tauri CLI is required to build the application:

    ```bash
    cargo install tauri-cli
    ```

    Verify the installation:
    ```bash
    cargo tauri --version
    ```
  </Step>

  <Step title="Platform-Specific Dependencies">
    Install platform-specific build dependencies:

    <CodeGroup>
    ```bash Ubuntu/Debian
    sudo apt update
    sudo apt install libwebkit2gtk-4.0-dev \
      build-essential \
      curl \
      wget \
      file \
      libssl-dev \
      libgtk-3-dev \
      libayatana-appindicator3-dev \
      librsvg2-dev
    ```

    ```bash Fedora
    sudo dnf install webkit2gtk4.0-devel \
      openssl-devel \
      curl \
      wget \
      file \
      libappindicator-gtk3-devel \
      librsvg2-devel
    ```

    ```bash Arch Linux
    sudo pacman -Syu
    sudo pacman -S webkit2gtk \
      base-devel \
      curl \
      wget \
      file \
      openssl \
      appmenu-gtk-module \
      gtk3 \
      libappindicator-gtk3 \
      librsvg
    ```

    ```bash macOS
    # Xcode Command Line Tools are required
    xcode-select --install
    ```

    ```powershell Windows
    # Visual Studio C++ Build tools or Visual Studio 2022
    # WebView2 is included in Windows 10/11
    # Download from: https://visualstudio.microsoft.com/downloads/
    ```
    </CodeGroup>
  </Step>
</Steps>

## Clone the Repository

Clone the Print My Bridge repository from GitHub:

```bash
git clone https://github.com/your-username/print-my-bridge.git
cd print-my-bridge
```

## Building the Application

### Development Build

For development and testing, use the dev command which provides hot-reloading:

<Steps>
  <Step title="Navigate to the Tauri directory">
    ```bash
    cd src-tauri
    ```
  </Step>

  <Step title="Build dependencies">
    ```bash
    cargo build
    ```

    This command downloads and compiles all Rust dependencies listed in `Cargo.toml`.
  </Step>

  <Step title="Run in development mode">
    ```bash
    cargo tauri dev
    ```

    This command:
    - Compiles the Rust backend
    - Loads the UI from the `ui/` directory
    - Opens the application with dev tools enabled
    - Enables hot-reloading for UI changes
  </Step>
</Steps>

### Production Build

For a production-ready build with optimizations:

<Steps>
  <Step title="Build the release version">
    ```bash
    cd src-tauri
    cargo tauri build
    ```

    This creates an optimized build with:
    - Level 3 optimizations (`opt-level = 3`)
    - Link-time optimization (LTO)
    - Stripped debug symbols
    - Smaller binary size
  </Step>

  <Step title="Locate the built artifacts">
    The built application will be in:

    <CodeGroup>
    ```bash macOS
    src-tauri/target/release/bundle/macos/Print My Bridge.app
    # Or DMG installer:
    src-tauri/target/release/bundle/dmg/Print My Bridge_*.dmg
    ```

    ```bash Linux
    # Debian package:
    src-tauri/target/release/bundle/deb/print-my-bridge_*.deb
    # AppImage:
    src-tauri/target/release/bundle/appimage/print-my-bridge_*.AppImage
    ```

    ```powershell Windows
    # MSI installer:
    src-tauri\target\release\bundle\msi\Print My Bridge_*.msi
    # Executable:
    src-tauri\target\release\print-my-bridge.exe
    ```
    </CodeGroup>
  </Step>
</Steps>

## Build Configuration

The build process is configured through several files:

### Cargo.toml

The main Rust configuration file defines:
- Package metadata (name, version, authors)
- Dependencies and their versions
- Build optimizations
- Platform-specific dependencies

Key dependencies:
- **tokio** (1.0): Async runtime
- **warp** (0.3): HTTP server framework
- **tauri** (2.0): Desktop application framework
- **serde_json** (1.0): JSON serialization

### tauri.conf.json

Configures the Tauri application:
- Application identifier and metadata
- Window configuration
- Bundle settings for each platform
- Security permissions and capabilities

### Build Profile

The release profile in `Cargo.toml` optimizes for:
```toml
[profile.release]
opt-level = 3        # Maximum optimizations
lto = true          # Link-time optimization
codegen-units = 1   # Single codegen unit for better optimization
panic = "abort"     # Smaller binary size
strip = true        # Remove debug symbols
```

## Testing the Build

After building, verify the application works correctly:

### Run Tests

```bash
cd src-tauri
cargo test
```

### Run the Application

<CodeGroup>
```bash Development Mode
cargo tauri dev
```

```bash Release Binary
# macOS
./target/release/print-my-bridge

# Linux
./target/release/print-my-bridge

# Windows
.\target\release\print-my-bridge.exe
```
</CodeGroup>

### Verify Functionality

1. Check that the application starts without errors
2. Verify the system tray icon appears
3. Test the configuration interface
4. Confirm the HTTP server starts on the configured port
5. Test printing functionality with a sample document

## Troubleshooting Build Issues

### Rust Toolchain Issues

If you encounter Rust version issues:
```bash
rustup update stable
rustup default stable
```

### Missing Dependencies

If build fails with linker errors, ensure all platform dependencies are installed. See the Prerequisites section above.

### Cargo Build Cache

Clear the build cache if you encounter strange errors:
```bash
cargo clean
cargo build
```

### Tauri CLI Issues

Reinstall the Tauri CLI:
```bash
cargo uninstall tauri-cli
cargo install tauri-cli
```
