Print My Bridge is available for Windows, macOS, and Linux. You can install it using pre-built binaries or build from source.

## System requirements

Before installing Print My Bridge, ensure your system meets these requirements:

<CardGroup cols={3}>
  <Card title="Windows" icon="windows">
    Windows 10 or later
  </Card>
  
  <Card title="macOS" icon="apple">
    macOS 10.13 (High Sierra) or later
  </Card>
  
  <Card title="Linux" icon="linux">
    Modern Linux distribution (64-bit)
  </Card>
</CardGroup>

<Note>
Print My Bridge requires an active printer connected to your system. You can verify printer availability after installation.
</Note>

## Pre-built binaries

The easiest way to install Print My Bridge is to download pre-built binaries from the releases page.

<Steps>
  <Step title="Download the installer">
    Visit the [Releases](https://github.com/your-username/print-my-bridge/releases) page and download the installer for your platform:
    
    - **Windows**: `Print My Bridge_x.x.x_x64_en-US.msi`
    - **macOS**: `Print My Bridge.app` (Universal binary for Intel and Apple Silicon)
    - **Linux**: `print-my-bridge_x.x.x_amd64.deb`
  </Step>
  
  <Step title="Install the application">
    Run the installer for your platform:
    
    <Tabs>
      <Tab title="Windows">
        Double-click the `.msi` file and follow the installation wizard. Print My Bridge will be installed to:
        
        ```
        C:\Program Files\Print My Bridge\
        ```
        
        The installer will create a desktop shortcut and start menu entry.
      </Tab>
      
      <Tab title="macOS">
        Extract the `.app` file and drag it to your Applications folder:
        
        ```bash
        # Mount the DMG (if applicable)
        open Print\ My\ Bridge.dmg
        
        # Copy to Applications
        cp -r Print\ My\ Bridge.app /Applications/
        ```
        
        <Warning>
        On first launch, macOS may show a security warning because the app is from an unidentified developer. Right-click the app and select "Open" to bypass this warning.
        </Warning>
      </Tab>
      
      <Tab title="Linux">
        Install the `.deb` package using your package manager:
        
        ```bash
        # Debian/Ubuntu
        sudo dpkg -i print-my-bridge_x.x.x_amd64.deb
        sudo apt-get install -f  # Install dependencies
        
        # Or use apt directly
        sudo apt install ./print-my-bridge_x.x.x_amd64.deb
        ```
        
        For other distributions, you can build from source (see below).
      </Tab>
    </Tabs>
  </Step>
  
  <Step title="Launch Print My Bridge">
    Start the application from your applications menu, start menu, or by running:
    
    <CodeGroup>
      ```bash Windows
      # Start from Start Menu or run
      "C:\Program Files\Print My Bridge\Print My Bridge.exe"
      ```
      
      ```bash macOS
      open -a "Print My Bridge"
      ```
      
      ```bash Linux
      print-my-bridge
      ```
    </CodeGroup>
  </Step>
</Steps>

## Build from source

If you prefer to build Print My Bridge from source or pre-built binaries aren't available for your platform, follow these steps.

<Warning>
Building from source requires Rust 1.70+ and familiarity with command-line tools.
</Warning>

<Steps>
  <Step title="Install Rust and dependencies">
    First, install Rust and the Tauri CLI:
    
    ```bash
    # Install Rust
    curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
    
    # Restart your terminal, then install Tauri CLI
    cargo install tauri-cli
    ```
    
    <Tabs>
      <Tab title="Linux">
        Install additional dependencies:
        
        ```bash
        # Debian/Ubuntu
        sudo apt update
        sudo apt install libwebkit2gtk-4.0-dev \
          build-essential \
          curl \
          wget \
          file \
          libssl-dev \
          libgtk-3-dev \
          libayatana-appindicator3-dev \
          librsvg2-dev
        
        # Fedora
        sudo dnf install webkit2gtk4.0-devel \
          openssl-devel \
          curl \
          wget \
          file \
          libappindicator-gtk3-devel \
          librsvg2-devel
        
        # Arch
        sudo pacman -S webkit2gtk \
          base-devel \
          curl \
          wget \
          file \
          openssl \
          appmenu-gtk-module \
          gtk3 \
          libappindicator-gtk3 \
          librsvg
        ```
      </Tab>
      
      <Tab title="macOS">
        No additional dependencies required beyond Xcode Command Line Tools:
        
        ```bash
        xcode-select --install
        ```
      </Tab>
      
      <Tab title="Windows">
        Install Visual Studio Build Tools:
        
        1. Download [Visual Studio Build Tools](https://visualstudio.microsoft.com/downloads/)
        2. Install "Desktop development with C++" workload
        3. Install [WebView2](https://developer.microsoft.com/en-us/microsoft-edge/webview2/) runtime
      </Tab>
    </Tabs>
  </Step>
  
  <Step title="Clone the repository">
    Clone the Print My Bridge repository:
    
    ```bash
    git clone https://github.com/your-username/print-my-bridge.git
    cd print-my-bridge
    ```
  </Step>
  
  <Step title="Build the application">
    Build Print My Bridge:
    
    ```bash
    cd src-tauri
    cargo tauri build
    ```
    
    This will:
    - Compile the Rust backend
    - Bundle the frontend assets
    - Create a platform-specific installer
    
    The build process may take 5-10 minutes on the first run.
  </Step>
  
  <Step title="Find the installer">
    After building, find your installer in:
    
    ```bash
    src-tauri/target/release/bundle/
    ```
    
    - **Windows**: `nsis/Print My Bridge_x.x.x_x64-setup.exe` or `msi/Print My Bridge_x.x.x_x64_en-US.msi`
    - **macOS**: `macos/Print My Bridge.app` or `dmg/Print My Bridge_x.x.x_universal.dmg`
    - **Linux**: `deb/print-my-bridge_x.x.x_amd64.deb` or `appimage/print-my-bridge_x.x.x_amd64.AppImage`
  </Step>
</Steps>

## First launch

When you launch Print My Bridge for the first time, the application will:

<Steps>
  <Step title="Create configuration file">
    A default configuration file is created at:
    
    - **Windows**: `%APPDATA%\print-my-bridge\print-my-bridge.toml`
    - **macOS**: `~/Library/Application Support/print-my-bridge/print-my-bridge.toml`
    - **Linux**: `~/.config/print-my-bridge/print-my-bridge.toml`
  </Step>
  
  <Step title="Start HTTP server">
    The HTTP server starts automatically on `http://127.0.0.1:8765`
  </Step>
  
  <Step title="Show configuration window">
    The main window displays your configuration and allows you to generate an API token
  </Step>
</Steps>

<Tip>
Print My Bridge minimizes to the system tray by default. Click the tray icon to show or hide the window.
</Tip>

## Verify installation

To verify that Print My Bridge is running correctly, check the health endpoint:

```bash
curl http://localhost:8765/health
```

You should receive a response like:

```json
{
  "status": "ok",
  "service": "print-my-bridge",
  "version": "0.1.0"
}
```

<Note>
If you receive a "Connection refused" error, ensure Print My Bridge is running. Check the system tray for the application icon.
</Note>

## Configuration

The configuration file uses TOML format. Here's the default configuration:

```toml print-my-bridge.toml
# Server configuration
host = "127.0.0.1"
port = 8765
max_file_size_mb = 50
rate_limit_per_minute = 60

# Application settings
auto_start = false
minimize_to_tray = true

# Security settings
allowed_origins = ["*"]
allowed_file_types = ["pdf", "html", "text", "image"]
default_printer = ""
```

You can edit this file manually or use the application interface to update settings.

## Auto-start configuration

To launch Print My Bridge automatically when your system starts:

1. Open Print My Bridge
2. Navigate to Settings
3. Enable "Start on system startup"
4. Restart your computer to test

Alternatively, edit the configuration file and set:

```toml
auto_start = true
```
